# カット再修正 + 元素材音声復元 報告

対象: `reference/reference.mp4` を正として再照合(既存レポート・編集計画は正解扱いしない)
出力: `output/prototype_cut_audio_rebuilt.mp4`(2450フレーム / 81.666667秒 / 30fps / 540×960 / H.264 / AAC-LC 48kHz mono)
スクリプト: `scripts/render_cut_audio_rebuilt.py`

## 1. 判明した映像の問題(旧 prototype_cut_fixed.mp4)

15.133〜21.5秒を素材15のミドルショットだけで構成していたが、見本は
**別アングル(ズームクロップ)を交互に挟む構成**だった。素材15には
「しゃがんで取り出す」動作が2回あり(1回目 6.8〜10.2秒、2回目 10.5〜14.8秒)、
旧版は 18.100秒で1回目のしゃがみ(素材15@8.3)、19.267秒で2回目のしゃがみ
(素材15@13.3)へ跳んでいたため、同じ店員が二度屈み、ボトルを持つ→置く→
また持つように見える重複が生じていた。

テンプレート照合(グレースケールNCC、スコア0.95〜0.9999)で見本の各ショットを
全素材に対して照合した結果:

- 見本15.13〜16.53の棚アップは **素材18のズームクロップ**(未使用扱いだった素材。
  「棚が不一致」は誤判定で、二階堂・KAHLUA・白桃の並びが一致、score 0.996)
- 見本18.10〜19.27は素材15の「ボトルを置き手が離れる」箇所のズームクロップ
- 見本19.27〜20.73のミドルは素材15の **2回目の起き上がり(13.367秒〜)のみ** を使用
  (1回目のしゃがみは見本に存在しない)
- 見本20.73〜21.50はお茶ボトルのズームクロップ(キャップは開けない)
- 冷蔵庫(素材13)は0.85秒から連続2.633秒。**取り出すのは冷えたグラス**
- 氷(素材1)は0.433秒から。蓋開け→スコップ→27.067秒で見本と同じパンチイン
  (時間は連続、フレーミングのみ拡大)に切り替え
- 28.733秒からの引き(素材6)は1.583秒開始。旧値0.9のままだと後続カットで
  0.3秒の巻き戻り再生が発生していた

## 3. 音声の再構築

旧版は会話ステム(prototype_text 由来)のみで、16.0〜40.5 / 57.0〜60.0 /
63.0〜65.5 / 74.5〜80.0 秒が完全無音(実測 -104〜-240dBFS)だった。

方式:
- 会話ステム = `prototype_cut_fixed.mp4` の音声をそのまま使用(絶対時刻・内容維持)
- 全46セグメントについて、映像と同じ素材・同じ source_in/out から
  オリジナル音声を切り出して現場音トラックを構成(48kHz mono化、両端12ms
  コサインフェードでクリック防止)
- 会話ステムの有音区間(RMS>-55dBFS、前後0.12秒拡張)では現場音を **-23dB に
  ダッキング**(120msスムージング)して二重会話を防止。会話の合間は等倍で復元
- 黒画面(70.1〜72.267秒)は直前素材12の音声を2.367秒から途切れず継続(×0.5)
- クリップ回避は現場音側のみ ×0.819(会話レベル不変)

復元された現場音(素材の実収録音、効果音の新規生成なし):
- 棚のボトルを取る音・置く音(素材18/15、15.1〜21.5秒)
- 冷蔵庫の開閉音・庫内から取り出す音(素材13、21.5〜24.1秒)
- ストッカーの蓋、スコップ、氷がグラスに入る音(素材1、24.1〜28.7秒)
- キャップ開閉・焼酎とお茶を注ぐ音・グラス操作(素材6、28.7〜40.5秒)
- 混ぜる音・提供音・店内environ(素材8/14/17/20/2/21/16/5/7/9、39.4〜70.1秒)
- 足音・ドア・廊下の環境音(素材12/23/24、68.9〜81.7秒)
